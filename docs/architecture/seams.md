# Skygest Seams Inventory

A seam is a place where one component hands data to another and the two could in principle be swapped independently. **Layer-level seams** are Effect `ServiceMap.Service` Tags: producer depends on the Tag, Layer implementation is swappable inside one runtime. **Transport-level seams** cross a process, disk, or Worker boundary — Service Bindings, D1 rows, R2 objects, HTTP endpoints, MCP tools, filesystem frontmatter, tsconfig `paths` aliases. This document is the inventory; `system-context.md` is the subsystem map and `resolution-trace.md` is the walkthrough of how one post crosses these seams. Tag literals and Schema names are quoted verbatim so they are grepable.

## Stability heat map

Ordered by blast radius, highest first.

**1. `@skygest/domain/*` tsconfig `paths` alias.** One string in `skygest-editorial/tsconfig.json` pins the editorial repo to the sibling `skygest-cloudflare/src/domain/` tree. Every shared Schema crosses it — `Did`, `PostUri`, `StoryFrontmatter`, `EditorialPickBundle`, `Variable`, `Series`, `Distribution`. Every editorial script and skill imports through it; every cache manifest decodes through it. Breaking it breaks every `bun scripts/*.ts` call at once, including `build-graph`. **Defend now.**
