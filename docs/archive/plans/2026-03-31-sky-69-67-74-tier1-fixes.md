### Task 1: SKY-74 — MCP numeric parameter coercion

The MCP protocol sends all tool params as JSON. LLM clients often serialize numbers as strings (`"20"` instead of `20`). The MCP input schemas in `src/domain/bi.ts`, `src/domain/editorial.ts`, and `src/domain/curation.ts` use `Schema.Number` which rejects string-encoded numbers.

**Files:**
- Modify: `src/domain/bi.ts`
- Modify: `src/domain/editorial.ts`
- Modify: `src/domain/curation.ts`
- Create: `tests/mcp-input-coercion.test.ts`

**Step 1: Write the failing test**

Create `tests/mcp-input-coercion.test.ts`:

**Step 2: Run test to verify it fails**

Run: `bun run test tests/mcp-input-coercion.test.ts`
Expected: FAIL — `Schema.Number` rejects string inputs.

**Step 3: Implement the coercion**

The fix: create a `FlexibleNumber` helper that accepts both `number` and `string` (parsing string to number). Use `Schema.Union` of `Schema.Number` and `Schema.NumberFromString`.

In `src/domain/bi.ts`, add near the top:

```typescript
/** Accepts both number and string-encoded number (for MCP protocol compatibility). */
const FlexibleNumber = Schema.Union(Schema.Number, Schema.NumberFromString);
```

Then replace every `Schema.Number` in MCP-facing input schemas with `FlexibleNumber`, preserving annotations:

- `SearchPostsInput`: `since`, `until`, `limit`
- `GetRecentPostsInput`: `since`, `until`, `limit`
- `GetPostLinksInput`: `since`, `until`, `limit`
- `ListExpertsInput`: `limit`
- `ListExpertsInput`: `active` — also needs `Schema.Union(Schema.Boolean, Schema.BooleanFromString)` since it has the same MCP serialization issue
- `GetPostThreadInput`: `depth`, `parentHeight` (keep `.pipe(Schema.int(), Schema.between(...))` after)
- `GetThreadDocumentInput`: `depth`, `parentHeight`, `maxDepth`, `minLikes`, `topN`

In `src/domain/editorial.ts`:
- Import `FlexibleNumber` from `"./bi"` (or define locally)
- `ListEditorialPicksInput`: `since`, `limit` → `FlexibleNumber`
- `minScore` → use `Schema.Union(EditorialScore, Schema.compose(Schema.NumberFromString, EditorialScore))`

In `src/domain/curation.ts`:
- `ListCurationCandidatesInput`: `since`, `limit` → `FlexibleNumber`
- `minScore` → use `Schema.Union(CurationSignalScore, Schema.compose(Schema.NumberFromString, CurationSignalScore))`

**Important:** Do NOT change the internal query schemas (`SearchPostsQueryInput`, `GetRecentPostsQueryInput`, `GetPostLinksQueryInput`, `GetCuratedFeedInput`) — those are used internally with already-parsed numbers.

**Step 7: Write the integration test**

Create `tests/embed-type-ingest.test.ts` that proves embed type flows through ingest and gets persisted:

```typescript
import { SqlClient } from "@effect/sql";
import { Effect } from "effect";
import { describe, expect, it } from "@effect/vitest";
import { KnowledgeRepo } from "../src/services/KnowledgeRepo";
import { makeSqliteLayer } from "./support/runtime";
import { runMigrations } from "../src/db/migrate";

const makePost = (uri: string, embedType: string | null) => ({
  uri,
  did: "did:plc:test" as any,
  cid: null,
  text: "test post",
  createdAt: Date.now(),
  indexedAt: Date.now(),
  hasLinks: false,
  status: "active" as const,
  ingestId: `${uri}:create:none:${Date.now()}`,
  embedType,
  topics: [],
  links: []
});

describe("embed type persisted during ingest", () => {
  it.effect("stores embed_type=img for image posts", () =>
    Effect.gen(function* () {
      yield* runMigrations;
      const repo = yield* KnowledgeRepo;
      yield* repo.upsertPosts([makePost("at://did:plc:test/app.bsky.feed.post/img1", "img")]);

      const sql = yield* SqlClient.SqlClient;
      const rows = yield* sql<{ embed_type: string | null }>`
        SELECT embed_type FROM posts WHERE uri = ${"at://did:plc:test/app.bsky.feed.post/img1"}
      `;
      expect(rows[0]?.embed_type).toBe("img");
    }).pipe(Effect.provide(makeSqliteLayer()))
  );

  it.effect("stores embed_type=link for link card posts", () =>
    Effect.gen(function* () {
      yield* runMigrations;
      const repo = yield* KnowledgeRepo;
      yield* repo.upsertPosts([makePost("at://did:plc:test/app.bsky.feed.post/link1", "link")]);

      const sql = yield* SqlClient.SqlClient;
      const rows = yield* sql<{ embed_type: string | null }>`
        SELECT embed_type FROM posts WHERE uri = ${"at://did:plc:test/app.bsky.feed.post/link1"}
      `;
      expect(rows[0]?.embed_type).toBe("link");
    }).pipe(Effect.provide(makeSqliteLayer()))
  );

  it.effect("stores embed_type=null for text-only posts", () =>
    Effect.gen(function* () {
      yield* runMigrations;
      const repo = yield* KnowledgeRepo;
      yield* repo.upsertPosts([makePost("at://did:plc:test/app.bsky.feed.post/text1", null)]);

      const sql = yield* SqlClient.SqlClient;
      const rows = yield* sql<{ embed_type: string | null }>`
        SELECT embed_type FROM posts WHERE uri = ${"at://did:plc:test/app.bsky.feed.post/text1"}
      `;
      expect(rows[0]?.embed_type).toBeNull();
    }).pipe(Effect.provide(makeSqliteLayer()))
  );
});
```
