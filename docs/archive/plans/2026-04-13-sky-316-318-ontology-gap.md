## 4. SKY-318 gap table + 5 (recommended: 6) CQ drafts

**None of the required CQs exist.** Drafted below using the file's existing YAML schema.

### CQ-069 ‚Äî Dataset ‚Üí Series enumeration (must_have, enumerative)

Natural language: "What Series belong to a given Dataset?"

SPARQL:

```sparql
PREFIX sevocab: <https://skygest.dev/vocab/energy/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
SELECT ?dataset ?series ?seriesLabel WHERE {
  ?dataset a sevocab:EnergyDataset ;
           sevocab:hasSeries ?series .
  ?series a sevocab:Series ;
          rdfs:label ?seriesLabel .
}
```

Direct `hasSeries` enumeration. Structural twin of CQ-065.

### CQ-070 ‚Äî Series ‚Üí Variable lookup (must_have, relational)

"Which Variable does a given Series implement?"

```sparql
PREFIX sevocab: <https://skygest.dev/vocab/energy/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
SELECT ?series ?variable ?variableLabel WHERE {
  ?series a sevocab:Series ;
          sevocab:implementsVariable ?variable .
  ?variable a sevocab:EnergyVariable ;
            rdfs:label ?variableLabel .
}
```

Required axiom anchor: `implementsVariable owl:FunctionalProperty`.

### CQ-071 ‚Äî chain-agreement (must_have, integrity) ‚Äî the hard one

"Does the Dataset ‚Üí Series ‚Üí Variable path agree with the higher-level Dataset ‚Üí Variable view?"

```sparql
PREFIX sevocab: <https://skygest.dev/vocab/energy/>
SELECT DISTINCT ?dataset ?variable WHERE {
  {
    ?dataset sevocab:hasVariable ?variable .
  }
  UNION
  {
    ?dataset sevocab:hasSeries ?series .
    ?series  sevocab:implementsVariable ?variable .
  }
  FILTER EXISTS { ?dataset sevocab:hasVariable ?variable . }
  FILTER EXISTS { ?dataset sevocab:hasSeries/sevocab:implementsVariable ?variable . }
}
```

**Entailment-agnostic**: UNION + both-paths `FILTER EXISTS`. Does not rely on `owl:propertyChainAxiom`. Matches runtime union-merge semantics in `src/resolution/dataLayerRegistry.ts:570-604`.

### CQ-072 ‚Äî dedup/collision semantics (should_have, quantitative)

"When a single Variable is implemented by more than one Series, how many distinct publishing Datasets reach it?"

```sparql
PREFIX sevocab: <https://skygest.dev/vocab/energy/>
SELECT ?variable
       (COUNT(DISTINCT ?series)  AS ?seriesCount)
       (COUNT(DISTINCT ?dataset) AS ?datasetCount)
WHERE {
  ?series a sevocab:Series ;
          sevocab:implementsVariable ?variable ;
          sevocab:publishedInDataset ?dataset .
}
GROUP BY ?variable
HAVING (COUNT(DISTINCT ?series) > 1 OR COUNT(DISTINCT ?dataset) > 1)
```

Mirror of UC-DM-005 notes (lines 88‚Äì91) "set-union over all implementing Series".

### CQ-073 ‚Äî missing dataset-link guard (must_have, integrity)

"Are there any Series missing a dataset link?"

```sparql
PREFIX sevocab: <https://skygest.dev/vocab/energy/>
SELECT ?series WHERE {
  ?series a sevocab:Series .
  FILTER NOT EXISTS { ?series sevocab:publishedInDataset ?dataset . }
}
```

`expected_result: empty`. Read-side observability probe ‚Äî SHACL SeriesShape `minCount 1` is the enforcement side. Any row in the answer set is a schema failure.

### CQ-074 ‚Äî agent-scoped counts (must_have, quantitative)

"For a given Agent, how many Series and how many distinct Variables are published through the Dataset ‚Üí Series ‚Üí Variable chain?"

```sparql
PREFIX sevocab: <https://skygest.dev/vocab/energy/>
PREFIX dct: <http://purl.org/dc/terms/>
SELECT ?agent
       (COUNT(DISTINCT ?series)   AS ?seriesCount)
       (COUNT(DISTINCT ?variable) AS ?variableCount)
WHERE {
  ?dataset a sevocab:EnergyDataset ;
           dct:publisher ?agent ;
           sevocab:hasSeries ?series .
  ?series sevocab:implementsVariable ?variable .
}
GROUP BY ?agent
```

**Numbering note**: ticket says 5 CQs; natural drafting produces 6. If strictly 5, drop CQ-072 ‚Äî but that loses explicit ontology observability for multi-Series-per-Variable (see open question 1).
